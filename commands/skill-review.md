# /skill:review — Review Skill Improvement Candidates

Review and act on skill improvement candidates generated by the skill evolution engine.

---

## Instructions

1. **List candidates**: Read all `.candidate.md` files in `~/.claude/skills/.candidates/`
2. **For each candidate** (ordered by priority: high > medium > low):
   a. Show the skill name, priority, reason, and metrics
   b. Read the current SKILL.md for that skill
   c. Search Context7 for updated documentation on the skill's technology
   d. Propose specific improvements (updated keywords, revised content, new patterns)
3. **Ask the user** for each candidate:
   - **Accept**: Apply the improvements to the active SKILL.md, update REGISTRY.md keywords if needed, delete the candidate file
   - **Dismiss**: Delete the candidate file (no changes)
   - **Investigate**: Deep-dive into the skill's performance data in `logs/skill-stats.json` and `logs/skill-outcomes.jsonl`

If no candidates exist, run `node ~/.claude/scripts/skill-improver.js` first to generate them.

## Arguments

- No args: review all pending candidates
- `accept <name>`: auto-accept candidate for skill `<name>`
- `dismiss <name>`: dismiss candidate for skill `<name>`
- `generate`: force-generate new candidates from current data
